# 對話流程

## 使用者問一個問題後，系統做了什麼？

```
使用者：「本月營收多少？」
        │
        ▼
┌─────────────────────────────┐
│  1. 送出訊息                │
│  畫面顯示使用者的問題氣泡    │
│  同時送到後端處理            │
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│  2. 找到這個客戶的資料庫     │
│  每個客戶有自己的 DB         │
│  系統知道這個人屬於哪間公司   │
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│  3. 讀資料庫結構             │
│  系統讀取這間公司的所有表格   │
│  和欄位的中文名稱            │
│                             │
│  例如：                      │
│  「orders 表 = 訂單」        │
│  「total_amount = 訂單金額」  │
│                             │
│  薪資、銀行帳號等敏感欄位     │
│  → 直接排除，AI 看不到       │
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│  4. 問 AI                    │
│                              │
│  把以下東西一起送給 AI：       │
│  • 資料庫結構（中文名稱）     │
│  • 之前的對話紀錄（最多10輪） │
│  • 使用者的問題               │
│                              │
│  AI 會回傳：                  │
│  • 一句 SQL 查詢語句          │
│  • 一句中文回覆模板           │
│  • 它覺得自己多有把握（0-100%）│
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│  5. 安全檢查                 │
│                              │
│  • 只允許「查詢」，不允許      │
│    刪除或修改資料             │
│  • 再次確認沒碰到敏感欄位     │
│                              │
│  沒通過 → 告訴使用者          │
│  「此資料受限，無法查詢」     │
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│  6. 判斷 AI 有多少把握        │
│                              │
│  高（95% 以上）               │
│  → 直接查資料庫，回傳結果     │
│                              │
│  中（70-95%）                 │
│  → 查資料庫，但附註           │
│    「建議確認」+ 可以看 SQL   │
│                              │
│  低（70% 以下）               │
│  → 不查！請使用者把問題       │
│    說得更清楚                 │
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│  7. 查資料庫                 │
│  用 AI 產生的 SQL 去          │
│  這間公司的 DB 撈資料         │
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│  8. 格式化結果               │
│                              │
│  數字 → NT$1,234,567         │
│  表格 → 欄位名 + 資料列      │
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│  9. 存對話紀錄               │
│  使用者說了什麼 + AI 回了什麼 │
│  存到資料庫，下次可以回看     │
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│  10. 畫面顯示結果            │
│                              │
│  • 數字 → 綠色大字           │
│  • 表格 → 有標題的表格       │
│  • 不確定 → 文字引導          │
│  • 錯誤 → 紅色錯誤訊息       │
│                              │
│  同時更新左邊的對話清單       │
└─────────────────────────────┘

        系統：「本月營收為 NT$231,891」
```

## 多輪對話怎麼運作？

使用者問完「本月營收多少」後追問「跟上個月比呢」，系統會：

1. 把前面的對話（最多 10 輪）一起送給 AI
2. AI 看到之前在聊營收，知道「上個月」是指上個月的營收
3. 產生正確的 SQL，不需要使用者重複說明

## 敏感資料怎麼保護？

兩道防線：

1. **第一道**：組 AI 提示時，薪資、銀行帳號等欄位根本不給 AI 看 → AI 不知道有這些欄位，自然不會查
2. **第二道**：萬一 AI 還是產出了碰到敏感欄位的 SQL → 系統攔截，不執行，回覆「此資料受限」

## 寫入操作（待開發）

未來使用者說「幫我新增一筆訂單」時，流程會多一步：

```
AI 產生 INSERT SQL
       │
       ▼
顯示操作摘要讓使用者確認
「即將新增：客戶 ABC、金額 NT$12,500」
       │
  ┌────┴────┐
  ▼         ▼
確認        取消
  │         │
執行 SQL   丟棄
回報結果   「操作已取消」
```

所有寫入都要使用者親手按確認，不會自動執行。
